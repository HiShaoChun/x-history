# 安装和测试指南

## 安装步骤

1. **打开 Chrome 扩展管理页面**
   - 在 Chrome 地址栏输入：`chrome://extensions/`
   - 或通过菜单：`更多工具` → `扩展程序`

2. **启用开发者模式**
   - 在页面右上角找到"开发者模式"开关
   - 打开它

3. **加载扩展**
   - 点击左上角的"加载已解压的扩展程序"按钮
   - 选择本项目的根目录（包含 manifest.json 的文件夹）
   - 点击"选择文件夹"

4. **确认安装成功**
   - 应该能看到"X History Recorder"出现在扩展列表中
   - 状态显示为"已启用"
   - Chrome 工具栏会显示扩展图标（蓝色书本）

## 功能测试

### 测试 1：基础记录功能

1. 打开 [x.com](https://x.com) 或 [twitter.com](https://twitter.com)
2. 登录你的账号
3. 打开浏览器开发者工具（F12）
4. 切换到 Console 标签页
5. 应该能看到：`[X History] Content script loaded`
6. 在时间线上缓慢向下滚动，每条推文停留 2-3 秒
7. 继续滚动，浏览 5-10 条推文

### 测试 2：查看统计信息

1. 点击 Chrome 工具栏上的扩展图标
2. 弹窗应该显示：
   - 总记录数
   - 已点开数量
   - 总停留时间
3. 如果刚开始测试，数值应该很小（因为需要停留 ≥1.5 秒才会记录）

### 测试 3：查看完整历史

1. 在弹窗中点击"查看完整历史"按钮
2. 应该打开一个新标签页，显示历史记录
3. 记录按天分组，每天内按小时分组
4. 每条记录显示：
   - 时间
   - 作者
   - 停留时长
   - 推文摘要
   - "查看原推"链接

### 测试 4：详情页打开标记

1. 在 X 时间线上，点击某条推文进入详情页
2. 等待 2 秒
3. 返回历史查看页面，刷新
4. 该推文应该显示"↗已点开"标记

### 测试 5：失焦暂停

1. 在 X 时间线上停留在某条推文
2. 切换到其他标签页或最小化浏览器
3. 等待 10 秒
4. 切回 X 标签页
5. 检查该推文的停留时长，应该不包括失焦的时间

### 测试 6：数据清除

1. 打开完整历史页面
2. 点击"清除所有数据"按钮
3. 确认操作
4. 页面应该显示"暂无浏览记录"
5. 重新浏览一些推文，记录应该重新开始

## 调试技巧

### 查看 Console 日志

1. 在 x.com 页面按 F12 打开开发者工具
2. Console 标签页会显示扩展的运行日志
3. 关键日志：
   - `[X History] Content script loaded` - 内容脚本已加载
   - 错误信息会显示在这里

### 查看 Background Service Worker

1. 在 `chrome://extensions/` 页面
2. 找到 X History Recorder
3. 点击"service worker"链接（在"检查视图"下）
4. 查看后台脚本的日志和错误

### 查看 IndexedDB 数据

1. 在 x.com 页面按 F12
2. 切换到 Application 标签页
3. 左侧菜单：Storage → IndexedDB → XHistoryDB
4. 可以查看 tweets 表中的所有记录

### 常见问题

**Q: 没有记录任何推文**
- 检查是否在 x.com 或 twitter.com 域名
- 确认每条推文停留时间 ≥ 1.5 秒
- 打开 Console 查看是否有错误

**Q: 统计信息显示"加载中..."**
- 后台脚本可能未完全初始化
- 刷新页面或重新打开弹窗

**Q: DOM 结构变化导致抓取失败**
- X.com 可能更新了前端
- 需要调整 content.js 中的选择器
- 关键选择器：
  - `article[data-testid="tweet"]` - 推文容器
  - `a[href*="/status/"]` - 推文链接
  - `[data-testid="tweetText"]` - 推文正文

## 性能监控

- 扩展每 100ms 检查一次视口
- 仅在页面聚焦时计时
- IndexedDB 操作异步执行，不阻塞页面
- 预期内存占用：< 50MB
- 预期 CPU 占用：< 1%
