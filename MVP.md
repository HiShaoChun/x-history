# X 浏览历史记录器 — MVP 文档（最终版）

## 1. 目标

记录我每天在 x.com 上**实际看了哪些帖子、在什么时间点看的**，并能事后按小时回看。

不做：搜索、统计图表、云同步、跨设备、多浏览器。

## 2. 平台

- Chrome 浏览器扩展（Manifest V3）
- 内容脚本注入 x.com，被动观察页面，不调用 X 官方 API

## 3. 核心定义

### 什么算"看了"

两种信号，分开记录：

| 信号 | 触发条件 | 含义 |
|------|----------|------|
| **浏览（view）** | 帖子在视口内可见、连续停留 ≥ 1.5 秒 | 在时间线上扫到并停留过 |
| **点开（open）** | 进入某条推文的详情页 | 更强的兴趣信号 |

同一条帖子可能既有 view 又有 open，二者并存。

### 停留计时规则

- 帖子在视口内才计时。
- **页面失焦 / 切到别的标签页时暂停计时**，回到页面再继续，避免挂机刷高数据。

### 去重规则

- 按 `tweetId` 去重，每条帖子一条主记录。
- 同一帖子反复滑进滑出：累加停留总时长、累加浏览次数，不新建记录。
- 记录"首次看到时间"和"最后看到时间"。

## 4. 记录的数据

每条帖子存：

- `tweetId` — 唯一标识
- `author` — 作者 handle（如 @xxx）
- `text` — 推文正文
- `url` — 帖子链接（点击可回原文）
- `firstSeenAt` / `lastSeenAt` — 时间点
- `dwellMs` — 累计停留时长（失焦时不累加）
- `viewCount` — 被滑到的次数
- `opened` — 是否点开过详情

## 5. 存储

- 本地 IndexedDB
- **无限累积**，不自动清理；提供**手动清除**入口（按时间段或全清）
- 纯本地，不上传任何地方

## 6. 查看结果

查看页面**按天 + 按整点小时聚合**展示：

```
2026-06-23 周二

  09:00 · 8 条
    09:32  @userA   ↗已点开   停留 4s   「正文摘要……」
    09:34  @userB              停留 2s   「正文摘要……」

  10:00 · 5 条
    10:15  @userC   ↗已点开   停留 8s   「正文摘要……」
```

- 帖子归入哪个小时桶，以 `firstSeenAt` 为准。
- 每个小时桶显示条数；正文显示摘要，点击跳回原帖。

## 7. 明确不做（划清边界）

- ❌ 媒体内容（图片/视频本体，正文仍存）
- ❌ 搜索、筛选、标签
- ❌ 数据可视化、统计报表
- ❌ 云同步、自动导出
- ❌ 自动清理 / 数据保留策略（仅手动清除）
- ❌ Firefox / Edge / 移动端

## 8. 已知风险 / 待观察点

1. **DOM 结构依赖**：x.com 改前端会导致抓取失效，需后续维护。
2. **虚拟滚动**：列表节点回收复用，去重与停留计时是实现核心难点。
3. **正文存储成本**：无限累积 + 存正文，长期数据量会增长（纯文本量不大，可接受）。
4. **1.5 秒阈值**为拍脑袋值，上线后可能微调。
5. **隐私**：纯本地，但本质是在记录自己的浏览行为，心里有数。
